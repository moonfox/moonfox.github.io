安装Dabr必要条件

Posted on 2009-10-14 In 玩转软件 Symbols count in article: 298 Reading time ≈ 1 mins.

在你安装dabr时，请确认已满足以下必要条件

  1. php版本 5.2＋因为要用到 json_decode() function)
  2. 确保主机支持 mcrypt 模块
  3. 确保主机支持 mod_rewrite 模块

偶在安装后运行时遇到

  1. Parse error: syntax error, unexpected '=', expecting ')' in /data/home/zz43d1/htdocs/dabr/common/user.php on line 52
    可能就是php版本低于5.2造成的，偶用的是万网的主机，为php4.4.9。
  2. 如果主机不支持 mcrypt 模块，可以用第三方方法解决。这个在网上已经有方案了。

Belongs_to选项中的primary_key

Posted on 2009-10-08 In 程序开发 Symbols count in article: 366 Reading time ≈ 1 mins.

在定义mode时给belongs_to加入了:primary_key选项，运行时报错，提示Unknown primary-key。去网上寻找答案，在rubyonrails的官方网站的关于rails2.3.3新特性介绍中看到有这么一段描述

:primary_key option for belongs_to for broader support of legacy schemas and those using a separate UUID primary key: belongs_to :employee, :primary_key => ‘SSN’, :foreign_key => ‘EMPID’

原来直到rails2.3.3时primary_key才被belongs_to支持，偶用的是rails2.3.2，难怪会出现Unknown primary-key错误。

Ubuntu Gedit 乱码

Posted on 2009-10-08 In 我爱Linux Symbols count in article: 235 Reading time ≈ 1 mins.

在ubuntu中用gedit打开gb2312的文件时，会出现乱码。

解决方法a

  1. 进入终端
  2. shell>gconf-editor
  3. 依次点击apps->gedit-2->preference->encodings
  4. 在auto_detected和show_in_menu两个list类型的键中加入GBK或GB18030（注意：要把这两个值调到列表最前面）

解决方法b
在终端录入 gedit –encoding=编码 filename也可以解决打开gb2312文件乱码问题

Ubuntu的下载加速器

Posted on 2009-10-05 In 我爱Linux Symbols count in article: 287 Reading time ≈ 1 mins.

在ubuntu中，当你通过源更新系统时，如果此源比较慢的话，可以考虑使用axel ，它可以进行分段下载，从而加快你的下载速度。本人在更新某个国外源软件时下载速度非常慢，大概10K左右，在使用axel后，下载速度为56K（本人ADSL 512），效果还是相当明显滴！
