fedora 安装显卡驱动

我爱Linux
  • fedora

这次主要介绍一下fedora N卡的驱动安装方法。

1、当前内核版本号要和你用yum install kernel-headers已经安装的最新的版本要一致
rpm -qa | grep ^kernel
kernel-devel-2.6.33.5-124.fc13.i686
kernel-headers-2.6.33.5-124.fc13.i686
kernel-2.6.33.5-124.fc13.i686
这三个的版本号应该一致，如果不一致，请升级到最新版 运行yum update kernel升级至最新版本

2、下载显卡驱动
查看显卡型号
lspci | grep VGA
01:00.0 VGA compatible controller: nVidia Corporation GT218 [GeForce 210] (rev a2) 说明是geForce 210
官方网址http://www.nvidia.cn/Download/index.aspx?lang=cn

3、要修改的文件
vim /etc/modprobe.d/blacklist.conf
添加以下行阻止 nouveau 模块的加载
blacklist nouveau

4、安装驱动
重新启动系统后，进入grup选择菜单，在选择的内核上按e进行编辑，最后面接上一个空格和一个数字3，如
kernel /vmlinuz-2.6.33.5-124.fc13.i686 ro root=UUID=ece1ac0a-ef69-4d71-988e-20eef00b2f20 rd_NO_LUKS rd_NO_LVM rd_NO_MD rd_NO_DM LANG=en_US.UTF-8 SYSFONT=latarcyrheb-sun16 KEYBOARDTYPE=pc KEYTABLE=us rhgb quiet 3
然后Enter完成编辑，用b进行引导启动。

进入显卡驱动所在文件夹运行：sh ./NVIDIA-Linux-x86-256.35.run，安装后重新启动

5、验证安装是否成功
lsmod | grep ^nvidia，出现nvidia XXXXX，则说明安装成功，如果没有输出任何信息，则说明安装失败。

安装过程中遇到的最常见的问题

没有找到内核程序所在位置，则说明你安装的内核与当前内核不符，或与显卡驱动内核不符，请保存这两者一至。
请不要手动指定内核位置 –kernel-source-path，如果手动指定，会出现Unable to load the kernel module ‘nvidia.ko’.错误。

如果你的系统是ubuntu，也遇到了上面的问题，请用上面的方法解决即可。
