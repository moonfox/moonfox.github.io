集成安装 apache 、subversion

我爱Linux
  • Subversion
  • apache

需要软件：
SVN: Subversion
HTTPD: Apache Httpd
APR: Apache Apr
APU: Apache Apr-Util
以上软件均为目前最新版本

1. 下载源代码：
到各自的官方网站下载相应的源代码的tar.gz包，解包后准备编译安装：

2. 编译源代码：
按下面的顺序编译源代码：
(1) Apache Apr

cd apr

./configure –prefix=/usr/local/apr

make

make install

  • (2) Apache Apr-Util

    cd apr-util

    ./configure –prefix=/opt/apache/apr-util –with-apr=/opt/apache/apr –with-berkeley-db=/BerkeleyDB.4.7

    make

    make install

(3) Apache Httpd

cd httpd-

./configure

–prefix=/usr/local/apache
–enable-cache
–enable-disk-cache
–enable-mem-cache
–enable-ssl
–enable-http
–enable-rewrite
–enable-so
–enable-dav

–with-apr=/usr/local/apr
–with-apr-util=/usr/local/apr-util

./configure –prefix=/opt/apache/apache2 –enable-cache –enable-disk-cache –enable-mem-cache –enable-ssl –enable-http –enable-rewrite –enable-so –enable-dav –with-apr=/opt/apache/apr –with-apr-util=/opt/apache/apr-util

make

make install

(4)启动apache
cd /usr/local/apache
./bin/apachectl start
最后在浏览器输入http://localhost
若出现It Works!恭喜你，安装成功

(5)加入到服务
cp /usr/local/apache/bin/apachectl /etc/init.d/httpd
chmod +x /etc/init.d/httpd
chkconfig –add httpd
重启

(6)附：
./configure //配置源代码树
–prefix=/usr/local/apache2 //体系无关文件的顶级安装目录PREFIX ，也就Apache的安装目录。
–enable-module=so //打开 so 模块，so 模块是用来提 DSO 支持的 apache 核心模块
–enable-mods-shared=all //编译全部的模板，对于不需要我们可以在httpd.conf去掉。
–enable-cache //支持缓存
–enable-file-cache //支持文件缓存
–enable-mem-cache //支持记忆缓存
–enable-disk-cache //支持磁盘缓存
–enable-static-support //支持静态连接(默认为动态连接)
–enable-static-htpasswd //使用静态连接编译 htpasswd - 管理用于基本认证的用户文件
–enable-static-htdigest //使用静态连接编译 htdigest - 管理用于摘要认证的用户文件
–enable-static-rotatelogs //使用静态连接编译 rotatelogs - 滚动 Apache 日志的管道日志程序
–enable-static-logresolve //使用静态连接编译 logresolve - 解析 Apache 日志中的IP地址为主机名
–enable-static-htdbm //使用静态连接编译 htdbm - 操作 DBM 密码数据库
–enable-static-ab //使用静态连接编译 ab - Apache HTTP 服务器性能测试工具
–enable-static-checkgid //使用静态连接编译 checkgid
–disable-cgid //禁止用一个外部 CGI 守护进程执行CGI脚本
–disable-cgi //禁止编译 CGI 版本的 PHP我们不再使用worker模式编译apache，worker模式和php貌似有一些不协调不稳定之处。所以使用了默认的perfork模式。

4.安装Subversion
(1)下载neon库，使svn支持webdav, https加密的链接.
地址：http://www.webdav.org
下载之后放到subversion安装目录下,并重命名为neon即可, subversion会自动监测并配置

(2)安装expat
一个XML解释器，如果没有安装，在编译subversion时会出现 /usr/bin/ld: cannot find -lexpat 错误
安装 expat、libexpat1-dev

(3) 正式安装开始
cd subversion
./configure
–prefix=/usr/local/subversion
–with-apxs=/usr/local/apache/bin/apxs
–with-apr=/usr/local/apr/bin/apr-1-config
–with-apr-util=/usr/local/apr-util/bin/apu-1-config
–with-ssl
–with-zlib (本人由于在配置时出现错误，没有安装)
–enable-maintainer-mode
make
make install

./configure –prefix=/opt/svn –with-apxs=/opt/apache/apache2/bin/apxs –with-apr=/opt/apache/apr/bin/apr-1-config –with-apr-util=/opt/apache/apr-util/bin/apu-1-config –with-ssl –enable-maintainer-mode

(4)设置 SVN_EDITOR
SVN_EDITOR=’”/usr/local/more editors/bin/xemacs”‘(加‘’可以避免由空格引起的错误)
在这里设置为 SVN_EDITOR=vim
export SVN_EDITOR

运行svn –version

在64位机子上安装时如果出现
make: * [subversion/libsvn_ra_neon/libsvn_ra_neon-1.la] Error 1
修改subversion/neon/src/Makefile，将CFLAGS = -g -O2 改成 CFLAGS = -fPIC -g -O2 (-g -O2 -prefer-pic 也可以这样替换)
